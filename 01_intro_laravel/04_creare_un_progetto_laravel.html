Installiamo e configuriamo Laravel

In questo ultimo capitolo della nostra introduzione prenderemo dimistichezza con il file di configurazione della nostra macchina virtuale, modificheremo il nostro file Homestead.yaml, il file di host del computer, e installeremo Laravel connettendoci alla nostra macchina virtuale in ssh, imparando a lanciare comandi unix dal nostro computer.

Modificare il file di host del computer

Per poter modificare il file di host da una macchina windows, ci converrà utilizzare un editor di testo. Questa è anche una buona occasione per decidere quali utilizzare, dato che per editare i file con estensione php potremo utilizzare i più disparati. La scelta è del tutto personale, io preferisco utilizzare Sublime Text per editare il codice html e css, mentre per i file php e Javascript uso Visual Studio Code. Su Linux l'ottimo Gedit è sempre una valida alternativa, ma avete a disposizione anche editor ed ide più forniti. Scegliete quello che più vi piace.
Fatta questa doverosa parentesi, dicevamo: per poter modificare il file di host da Windows ci occorrerà un editor di testo, una volta scaricato, clicchiamo con il tasto destro sulla sua icona ed avviamolo come amministratore di sistema, dopodichè andiamo ad aprire il file di host dal nostro editor. Perchè dobbiamo farlo? Perchè tutti i sistemi operativi non ci permettono di modificare questo file senza i permessi da amministratore. Su un sistema linux basterà lanciare da terminale il seguente comando:

sudo gedit /etc/hosts
Su Windows dovremo aprire il file che si trova al percorso: C:\Windows\System32\Drivers\etc\hosts.

Una volta aperto il file, inseriamo la seguente stringa: "[Indirizzo IP macchina virtuale] laravel.local".
Cosa stiamo facendo?
Stiamo sostanzialmente dicendo al nostro computer: "Sappi che, ogni volta che io sulla barra degli url di un browser digiterò la stringa 'laravel.local', significa che vorrò effettuare una richiesta alla macchina con indirizzo IP che ti ho fornito prima di questa stringa."
Come indirizzo ip dovremo fornire quello che abbiamo messo nel file Homestead.yaml.

Connettersi alla macchina virtuale tramite protocollo Secure Shell

Da terminale, o da git bash, rechiamoci nella cartella in cui si trova il file vagrantfile, e avviamo la macchina virtuale col solito comando vagrant up. Una volta che la macchina è avviata, connettiamoci ad essa tramite il comando: vagrant ssh.
Il comando appena digitato ci permette di avviare una sessione di connessione con un protocollo SSH (Secure Shell), tra il nostro computer e la macchina cliente. In questo caso la Homestead che abbiamo installato. In parole povere questo cosa significa? Che ora abbiamo di fronte un terminale da cui lanciare comandi unix, questi comandi però non verranno eseguiti sulla nostra macchina, ma su quella vagrant.

Installare Laravel e mappare le cartelle della macchina virtuale

Una volta avviata la sessione ssh, verifichiamo di trovarci nella cartella home/vagrant/. altrimenti navighiamo utilizzando i comandi unix cd [nome cartella], ls per mostrare la lista delle cartelle, pwd per verificare in che cartella ci troviamo, cd .. per risalire l'albero delle cartelle e tronare indietro di una posizione.
Una volta nella cartella giusta lanciamo il seguente comando: composer create-project laravel/laravel --prefer-dist Code/laravel. Si tratta di una installazione piuttosto pesante, quindi potrebbe andare avanti per parecchi minuti, fatevi un caffè e aspettate :).

Complimenti!

Non possiamo ancora dire di aver finito, ci manca un ultimo passaggio! Chiudiamo la connessione ssh tramite il comando exit. Torneremo nel terminale della nostra macchina fisica, da qui, lanciamo il comando vagrant halt per fermare la macchina virtuale. Apriamo il file Homestead.yaml e modifichiamolo come segue:

Se fate caso alla sezione "sites" vedrete come tramite i comandi "map:" e "to:" io stia dicendo alla macchina homestead, che la stessa stringa che io utilizzo nel mio file di host, fungerà da stringa che indicherà la cartella che funge da radice del progetto. Ogni volta che sul mio browser digiterò "laravel.local", il mio file di host saprà che voglio essere indirizzato verso quel determinato IP locale, a questo punto, la macchina homestead, vedendosi arrivare una richiesta con quella determinata stringa, sa che dovrà indirizzarmi verso il "sito" mappato dalla riga "to:".
NOTA BENE: poichè a volte il semplice reboot della macchina virtuale non fa "digerire" la nuova configurazione a vagrant, è sempre buona norma, una volta riavviata la macchina dopo una modifica del file Homestead, lanciare nuovamente il provisioning delle porte e dei siti tramite il comando: vagrant provision. Se doveste vedere nel prossimo passo che la richiesta non va a buon fine, vi consiglio di farlo, dovrebbe risolvere il problema.
Una volta fatta questa modifica siamo pronti, abbiamo installato Laravel e siamo pronti a farlo girare sulla nostra macchina. riavviamo il server di sviluppo, attendiamo che abbia completato le configurazioni, e una volta pronti, apriamo un tab del browser e cerchiamo: "laravel.local". Se tutto è andato come previsto, dovreste trovarvi di fronte questa pagina.
NOTA 2: su piattaforme Linux potrebbero verificarsi altri due problemi: il primo è che la pagina potrebbe restituire un errore 403 o 401, si tratta semplicemente di un problema di permessi dell'utente. per risolvere facilmente la cosa ci bastera lanciare da dentro la cartella laraproject il comando sudo chmod 776.
Abbiamo finalmente installato Laravel, abbiamo scelto gli editor di testo preferiti, e abbiamo un intero server di sviluppo configurato e pronto all'uso. Possiamo finalmente cominciare il viaggio che ci porterà a sviluppare la nostra applicazione, che descriveremo nel capitolo 2.
