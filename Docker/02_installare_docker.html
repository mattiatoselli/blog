Una premessa

Sebbene Docker sia perfettamente compatibile con tutti i SO, in questa guida utilizzaremo Linux, ed in particolare Ubuntu. Il motivo principale è dovuto al fatto che i container sono generalmente basati su distribuzioni minimali Linux, poichè ne sfruttano l'isolamento del kernel. Trovo che lavorare con lo stesso ambiente a livello mentale sia un toccasana, ma è una soluzione del tutto personale, nulla vieta di usare altro, i comandi cambiano solo in minima parte.

Installare Docker da terminale

Per prima cosa, è sempre buona norma aggiornare il database dei repository di apt. Lanciamo i comandi: 

                    
$ sudo apt update
$ sudo apt-get update
$ sudo apt autoremove
Per sicurezza, rimuoviamo anche le possibili vecchie versioni di Docker. Successivamente installiamo Docker, una volta completato il download, facciamo in modo che il servizio Docker parta ad ogni riavvio della macchina: 
                    
$ sudo apt-get remove docker docker-engine docker.io
$ sudo apt-get install docker.io
$ sudo systemctl start docker
$ sudo systemctl enable docker
Possiamo controllare la versione di Docker tramite: 
                    
$ docker -v
Possiamo verificare al percorso dei servizi che docker sia effettivamente attivo: 
                    
$ cd /lib/systemd/system/
$ lib/systemd/system ls -al
Tra tutti i vari servizi presenti nella lista, dovremmo vederne due: docker.socket e docker.io

Operazioni conclusive

Per evitare di avere problemi di permessi nell'esecuzione di Docker, conviene seguire i consigli di post installazione della documentazione ufficiale. Per cui, diamo ancora alcuni ordini al nostro sistema, creando un gruppo per docker, cui aggiungeremo il nostro utente, e garantendogli permessi da superuser. 

                    
$ sudo groupadd docker
$ sudo usermod -aG docker $USER

Conclusione

Abbiamo installato ed avviato il servizio docker.io sulla nostra macchina Linux, ora siamo pronti per cominciare a utilizzare dei container. Nel prossimo capitolo scaricheremo un container con un server Nginx, vi accederemo tramite ssh, lo distruggeremo, ricreeremo da zero, ne creeremo una versione customizzata tramite uno scompattabile da eseguire quando preferiamo.
