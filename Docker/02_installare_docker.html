Una premessa

Sebbene Docker sia perfettamente compatibile con tutti i SO, in questa guida utilizzaremo Linux, ed in particolare Ubuntu. Il motivo principale è dovuto al fatto che i container sono generalmente basati su distribuzioni minimali Debian based, di conseguenza lavorare con lo stesso ambiente a livello mentale ne giova, ma è una soluzione del tutto personale, nulla vieta di usare altro, i comandi non cambiano.

Installare Docker da terminale

Per prima cosa, è sempre buona norma aggiornare il database dei repository di apt. Lanciamo i comandi:
$ sudo apt update
$ sudo apt-get update
$ sudo apt autoremove
Per sicurezza, rimuoviamo anche le possibili vecchie versioni di Docker.
$ sudo apt-get remove docker docker-engine docker.io
Ora, installiamo Docker:
$ sudo apt-get install docker.io
Una volta completato lo scaricamento e l'installazione, dobbiamo fare in modo che il servizio docker parta ad ogni avvio del sistema.
$ sudo systemctl start docker
$ sudo systemctl enable docker
Possiamo controllare la versione di Docker con il comando:
$ docker -v
Per verificare che il servizio docker sia effettivamente attivo possiamo verificare al percorso dei servizi:
cd /lib/systemd/system/
ls -al
Tra tutti i vari servizi presenti nella lista, dovremmo vederne due:
docker.socket
docker.socket

Conclusione

Abbiamo installato ed avviato il servizio docker.io sulla nostra macchina Linux, ora siamo pronti per cominciare a utilizzare dei container. Nel prossimo capitolo scaricheremo un container con un server Nginx, vi accederemo tramite ssh, lo distruggeremo, ricreeremo da zero, ne creeremo una versione customizzata tramite uno scompattabile da eseguire quando preferiamo.
