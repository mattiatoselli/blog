In questo primo esempio scenderemo nel pratico e approfitteremo per familiarizzare con Docker. Impareremo a scaricare container, inizializzarli, resettarli e distruggerli.

Immagini e containers

Quando scarichiamo una "macchina docker", oppure ci viene passato un file contenente una macchina virtualizzata, stiamo parlando di una immagine. L'immagine è come un blueprint di un container, definisce come sarà tale macchina. Essa è residente nel nostro computer, a meno che non decidiamo di eliminarla. Immaginiamo per esempio di scaricare Nginx, una volta che abbiamo la nostra immagine di Nginx possiamo avviarne una istanza in ascolto su qualche porta dell'host, l'istanza è un container, ma possiamo decidere di avviarne anche più di una istanza, in quel caso la nostra immagine rimane sempre una (Nginx, appunto), ma avremo più containers con la stessa immagine. Per intenderci, è un po' come la differenza tra un oggetto ed una istanza di un oggetto nei paradigmi di programmazione ad oggetti.
Vediamo subito di cosa sto parlando.

Scaricare ed eseguire immagini Docker da DockerHub

DockerHub è un servizio di repository open source dove tutti possono caricare e scaricare immagini già pronte. Come potete ben immaginare, la soluzione migliore è partire da delle macchine standard, e poi creare ciò che serve a noi. Il sito è facilmente consultabile e tramite una barra di ricerca possiamo cercare cosa vogliamo. Iniziamo procurandoci una immagine di Nginx. Per qualunque problema vi consiglio di consultare la documentazione ufficiale. 

                      
$ docker pull nginx
Una volta completate tutte le operazioni di scaricamento, possiamo fare un rapido controllo delle immagini presenti nel nostro sistema tramite: 
                      
$ docker images -a

Dovremmo vederne solo una presente. Possiamo quindi iniziare a prendere dimistichezza con la sintassi dei comandi Docker. Una rapida occhiata alla documentazione dovrebbe essere sufficiente per iniziare, ma esistono alcuni trucchetti che si imparano con l'esperienza. Non ci addentreremo troppo nei tecnicismi riguardanti Nginx in questo corso, sarà oggetto dei prossimi capitoli, e forse, di futuri tutorial dedicati. Avviamo il container. 
                      
$ docker run --name nginx001 -d -p 8080:80 nginx
Stiamo dicendo a Docker di avviare un container dall'immagine nginx e chiamare tale container con un nome opzionale nginx001, e mappare la porta 80 (http) con una delle nostre porte libere, in questa caso la 8080. Una rapida visita tramite browser al nostro ip locale: http://localhost:8080, ci presenterà la pagina di benvenuto del server. Una volta verificato che il nostro container funziona, possiamo stopparlo tramite il comando: 
                         
$ docker stop nginx001
come risposta a questo comando ci verrà restituito il nome del container, se proviamo a reinviare la richiesta sulla porta 8080, vedremo che il server infatti non risponde.

Conclusione

In questo tutorial abbiamo fornito una prima conoscenza basilare di Docker, abbiamo preso confidenza col concetto di container ed immagine. Nel prossimo capitolo impareremo a mappare il nostro container con il filesystem del nostro computer. Per qualunque chiarimento, comunque, vi invito a consultare la documentazione ufficiale disponibile qui.
